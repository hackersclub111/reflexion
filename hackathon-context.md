# Reflexion — Hackathon Context

## Hackathon: Google Cloud Rapid Agent Hackathon
## Track: Arize (Observability & Self-Improvement)
## Deadline: June 11, 2026 @ 2:00pm PDT

## Judging Bias (Arize Track)
Judges are Arize employees (Director of Partner Solutions, Head of Solutions Strategy):
- They care DEEPLY about observability integration quality
- They want to see Phoenix used in novel ways, not just basic tracing
- Self-improvement loop = their product's ultimate use case
- They'll evaluate: trace quality, MCP usage depth, evaluation pipeline

## Winning Pattern for This Hackathon
1. **Visible intelligence** — agent reasoning must be shown, not hidden
2. **Improvement loop** — must demonstrate failure → reflection → success
3. **Deep partner integration** — MCP must be central, not bolt-on
4. **Real-world impact** — must solve a problem people actually have
5. **Professional polish** — dashboard, README, video must look production-grade

## Constraints
- Only Google Cloud AI + Arize tools allowed
- Must use Gemini 3 (not GPT, Claude, etc.)
- Must integrate Arize Phoenix MCP server
- Must be newly created (May 5 - June 11)
- Must be open-source with OSI-approved license
- Must have hosted URL + public GitHub + demo video

## Competition Analysis
With 11,398 participants, even split across 6 tracks = ~1,900 per track.
But Arize is niche (observability) — likely fewer participants than MongoDB/Elastic.
Our edge: deepest possible Arize integration + novel self-improvement concept.

## Demo Story Arc
1. **Problem**: "AI agents repeat the same mistakes forever"
2. **Failure**: Show agent making a mistake on a task
3. **Reflection**: Agent analyzes its own trace, identifies the issue
4. **Improvement**: Agent generates and applies a fix
5. **Success**: Agent nails the same task with better quality
6. **Metrics**: Dashboard shows quality score improvement over time
7. **Architecture**: Show how Phoenix MCP enables this
8. **Vision**: "Any agent can be wrapped with Reflexion to self-improve"
